  1. ANALYSE DES AKTUELLEN ZUSTANDS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Stärken:
  ✓ Grundstruktur (NavigationRail + Screens) ist solide
  ✓ Material 3 wird bereits genutzt
  ✓ Responsive Breakpoints (800px, 1100px) sind vorhanden
  ✓ Gute Farbkodierung für Status (grün/orange/rot)

Schwächen / Verbesserungspotenzial:
  ✗ KPI-Cards: Bunte Farbverläufe + farbige Schatten wirken zu konsumig,
    nicht enterprise-tauglich (erinnert eher an Gaming-Apps)
  ✗ NavigationRail: Dunkel-Navy (#0F2744) mit weißen Icons ist zu kontraststark,
    kein visuelles Hierarchiegefühl
  ✗ Kein einheitlicher Schriftsatz – Systemfont ohne gezielte Gewichtung
  ✗ Fehlende visuelle Tiefe im Dashboard (alles gleich groß/wichtig)
  ✗ Statistiken: Balkendiagramme sind zu primitiv (kein Chart-Package)
  ✗ Einstellungen: Tab-Leiste hat keinen klaren Trenner zum Content
  ✗ Keine konsistenten Abstände im gesamten App-Bereich
  ✗ KPI-Karten scrollen horizontal – auf breiten Bildschirmen suboptimal

  2. REFERENZ-ANWENDUNGEN (Orientierung)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  • SAP Fiori       → Klare Grid-Struktur, Horizon-Design-System, Blau als
                      einzige Akzentfarbe, sehr ruhige Oberfläche
  • Odoo 17         → Weißer Hintergrund, saubere Tabellen, dezenter Sidebar
                      in Hellgrau, KPIs als schlichte Zahlen-Kacheln
  • Linnworks       → Datenorientiert, keine Ablenkung, konsistente Icon-Größen
  • Katana MRP      → Modernes SaaS-Look, Teal-Akzent, großzügiges Whitespace
  • inFlow          → Klar strukturierte Seitenleiste, Badges statt Farb-KPIs

  Gemeinsamer Nenner professioneller WMS-Apps:
  → Ein einziger Akzentton + Neutrals
  → Daten stehen im Vordergrund, Design tritt zurück
  → Konsistente 8px-Spacing-Grid
  → Tabellen / Listen sind das Herzstück – sauber und lesbar
  → Status-Farben nur dort wo semantisch notwendig

  3. DESIGNPRINZIPIEN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  I.   CLARITY FIRST – Daten sind das Produkt, nicht das Design
  II.  ONE ACCENT – Nur eine Akzentfarbe, Rest Neutrals
  III. 8PX GRID – Alle Abstände sind Vielfache von 4 oder 8
  IV.  FLAT & BORDERED – Keine Schatten, klare Trennlinien
  V.   SEMANTIC COLOR – Farbe nur bei Status (OK/Warnung/Kritisch)
  VI.  HIERARCHY – Schriftgrößen und -gewichte zeigen Wichtigkeit

  5. TYPOGRAFIE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Empfohlene Schriftart: "Inter" (Google Fonts, kostenlos)
  Alternative:           System-UI / SF Pro auf Apple, Segoe auf Windows

  Warum Inter?
  → Speziell für Bildschirme entwickelt
  → Exzellente Lesbarkeit bei kleinen Größen (ideal für Tabellen)
  → In Odoo, Linear, Vercel, Stripe verwendet

  TYPOGRAFISCHE SKALA
  ┌────────────────────────────────────────────────────────────────┐
  │  Display   28px / 700 (w-bold)    – Selten, große Zahlen       │
  │  H1        22px / 700             – Seiten-Titel               │
  │  H2        18px / 600 (w-semibold)– Panel-Überschriften        │
  │  H3        15px / 600             – Abschnittsheader           │
  │  Body      14px / 400             – Fließtext, Tabelleninhalt  │
  │  Small     12px / 400             – Labels, Zeitstempel        │
  │  Micro     11px / 500             – Badges, Tags, Chips        │
  └────────────────────────────────────────────────────────────────┘

  Letter-Spacing: 0 für Body; +0.2px für Labels; -0.5px für Display-Zahlen

    - 3px linker Randbalken in Akzentblau

  App-Name / Branding-Bereich:

  7. KPI-KARTEN (Dashboard)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  NACHHER (2 Varianten zur Auswahl):

  VARIANTE A – "Flat Metric" (empfohlen, modern-minimalistisch)
  ┌──────────────────────────┐
  │  ↗ Gesamtprofit          │  ← Label oben, kleines Icon daneben, --text-muted
  │                          │
  │  € 12.450,00             │  ← 28px / bold, --text-primary
  │                          │
  │  ████████████░░ +14 %    │  ← Optionaler Trend-Indikator (grün/rot)
  └──────────────────────────┘
  Stil: Weißer Hintergrund, 1px Border (--border), 8px Radius
  Icon-Farbe: Nur semantisch (grün/orange/rot/blau), NICHT als Hintergrund
  Kein Schatten, kein Verlauf

  VARIANTE B – "Left-Accent Border"
  ┌─╔══════════════════════════╗
  │ ║  OFFENE BESTELLUNGEN     ║  ← 4px linker Rand in Akzentfarbe
  │ ║  24                      ║
  └─╚══════════════════════════╝
  Stil: Weißer Hintergrund, 4px linker Rand in Statusfarbe
  Gut lesbar, klassisch im ERP-Bereich (SAP-ähnlich)

  LAYOUT auf Dashboard:
  - Kein horizontales Scrollen!
  - Grid: 2-4 Spalten je nach Bildschirmbreite
    < 700px  → 2 Spalten
    700-1100 → 3 Spalten
    > 1100px → 4 Spalten (oder alle 7 in einer Reihe möglich)

  11. APP-BAR / TOP-HEADER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  VORHER:  Dunkle Navy AppBar mit Inventar-Icon + Titel

  NACHHER:  Weißer / Surface-farbener Header (oder gar keine AppBar)
  Begründung: Wenn die Sidebar bereits dunkel ist, braucht die TopBar keinen
              weiteren dunklen Block. Das wirkt bei Desktop-Apps moderner.

  Option A (Empfohlen – für Desktop):
    - Kein klassischer AppBar
    - Stattdessen: Seiten-Titel (H1) oben links im Content-Bereich
    - Rechts: Breadcrumb + Account-Avatar + Benachrichtigungs-Icon
    - Hintergrund des Header-Bereichs: --bg-app oder --bg-surface
    - Trennlinie unten: --border

  Option B (Mobile – bestehende AppBar angepasst):
    - Hintergrund: --nav-bg (#1E293B) – statt #0F2744, wärmer
    - Kein Icon im Titel (Logo gehört in die Sidebar)
    - Nur Seiten-Titel + Hamburger

  12. STATISTIKEN-SCREEN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Aktuell: Manuelle Container-Balken (keine Chart-Library)

  Empfehlung: Package "fl_chart" (pub.dev) hinzufügen
    - BarChart für Monats-Profit → sieht sofort professionell aus
    - LineChart für Trend-Verlauf
    - PieChart für Käufer-Anteile

  Alternativ ohne zusätzliches Package:
    - Bestehende Balken verfeinern: Abgerundete Enden, Hover-Tooltips,
      Grid-Linien, X/Y-Achsenbeschriftung
    - Werte direkt am Balken anzeigen (nicht nur rechts daneben)

  13. SPACING-SYSTEM (8px-Grid)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Alle Abstände sind Vielfache von 4px:

    4px  – Sehr eng, z.B. Icon-zu-Text
    8px  – Standard-Innen-Abstand
   12px  – Karten-Innenabstand (kompakt)
   16px  – Standard Padding (Seiten-Content, Card-Padding)
   20px  – Zwischen Sektionen
   24px  – Zwischen Cards / Panels
   32px  – Große Seitentrennungen

  Seitenrand: 24px auf Desktop, 16px auf Mobile

  Aktuell: Material Icons (gemischt Outlined + Filled)

  Empfehlung:
  → Konsequent OUTLINED Icons im inaktiven Zustand
  → FILLED Icons nur im aktiven/ausgewählten Zustand (bereits so begonnen)
  → Icon-Größen vereinheitlichen: 18px für Inline-Icons, 20px für Navigations-
    Icons, 24px für Haupt-Aktionen

  Paket-Empfehlung (optional): "lucide_icons" oder "phosphor_flutter"
  → Lucide Icons = modern, sehr konsistent, in Linear/Vercel/Figma verwendet
  → Würde dem App einen modernen SaaS-Charakter geben
